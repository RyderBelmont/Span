Setup

  • There is now an option for who to automatically open when first starting the application instead of automatically opening whoever was opened last.

Dialogue Editor

  • A "Winner" role is now available for filters, so you can make conditions based on the player who had the best hand.
  • ~clothing.toPlural~ is available as a variable which pluralizes the clothing name.

Writing Aid

  • The number of situations you've responded to is now listed. This number does not include any situations created prior to v3.1.

Bug Fixes

  • Respond to This no longer pulls up a previous response if the selected stages do not match. For example, if you make a response that by default applies to stages 1-5, and then filter that down to just stage 2, creating another response will make a new response in stages 1-5 rather than reopening the response in only stage 2.
  • Fixed issue where an item with position: "both" covering a bare chest and lower underwear (or vice versa) would be treated as a major removal rather than an important reveal when making responses.
  • Fixed crash when choosing a portrait for a reskin before actually creating the reskin's images.
  • Manually creating a variable test for something like Hand Quality and using a variable on the righthand side will no longer wipe out that value when coming back to the case.
  • Filtering on a target for a case with a gender or size requirement will now filter the target possibilities down to only those who meet the criteria.
  • Fixed issue where creating a new skin on a character that has not been opened previously in the current session would wipe out that character's .xml files.
